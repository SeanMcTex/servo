# Issue: Add Memory System for Past Messages

## Problem
Servo currently has no persistent memory of past conversations or messages. Users may want to recall previous responses from the virtual pet for reference or to understand the pet's perspective on various screen activities.

## Solution
Implement a system to:
1. Store past messages generated by Servo
2. Allow recall of previous messages
3. Integrate with existing chatrecall functionality

## Implementation Details
- Extend `AppState` to include message history tracking
- Add methods to store new messages
- Implement retrieval of past messages using chatrecall capabilities
- Consider persistence of message history across app sessions

## Acceptance Criteria
- Messages are stored when Servo generates utterances
- Past messages can be recalled and viewed
- System integrates with existing chatrecall extension
- Message history can persist between app sessions
